    4. Variable Assignment

  • A variable is a container or location in memory used to store some data.
  • We don't need to declare the variable before using it. You can simply assign a value to a variable and it exists.
  • The rules of writing variables are the same as the rules of writing identifiers in Python.
  • We don't have to declare the type of a variable; Python handles it internally according to the value assigned.
    • a = 5
    print(a)
    5

    NOTE:

    • Dynamic typing: The type of a variable is determined at run time. Example: a = 5
    • Static typing: The type of a variable is determined at declaration time. Example: int a = 5
    • Dynamic binding: It is not possible to decide which piece of code will be executed because of a method call at compile time.
    • Static binding: Binding that takes place at compile time is known as static binding. Example: int a = 5

    6. IDENTIFIERS

  • A Python identifier is a name used to identify a variable, function, class, module, or other object.
  • An identifier starts with a letter (A-Z, a-z) or an underscore (_) followed by zero or more letters, underscores, or digits (0 to 9).
  • The following are naming conventions for Python identifiers:
  • Class names start with an uppercase letter. All other identifiers start with a lowercase letter.
  • Starting an identifier with a single leading underscore indicates that the identifier is private.
  • Starting an identifier with two leading underscores indicates a strongly private identifier.
  • If the identifier also ends with two trailing underscores (__), it is a language-defined special name.
    • # Variable name 
name = 'bibek'
print(name)  # Output: bibek

# Variable name '_'
_ = 'bibek'
print(_)  # Output: bibek

# Variable name 'first_name'
first_name = 'bibek'
print(first_name)  # Output: bibek
